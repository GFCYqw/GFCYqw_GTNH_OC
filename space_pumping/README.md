# Space Pump — 太空电梯流体监控与维持系统

GTNH 太空钻机（Space Pump）自动化管理脚本集，基于 OpenComputers。

## 文件说明

| 文件 | 类型 | 说明 |
|------|------|------|
| `pump.lua` | **主脚本** | AR 眼镜监控 + 优先级多槽位分配 + 自动维持，推荐使用 |
| `pump_wiki.lua` | 参考 | GTNH 中文维基版，带 GPU 图形界面与手动控制 |
| `fluids_AR.lua` | 独立脚本 | 纯 AR 眼镜流体监控（无钻机控制） |
| `list_fluids.lua` | 占位 | 空文件 |

## pump.lua v2.0 — 主脚本

### 功能

1. **自动发现钻机** — 扫描 `gt_machine` 组件，正则匹配 `projectmodulepumpt[1-3]` 检测等级
2. **等级显示** — AR 眼镜和终端均显示泵等级（LV1/LV2/LV3）与槽位数
3. **优先级多槽位分配** — 按 `FLUID_CONFIGS` 顺序分配不同流体到不同槽位
4. **200% 目标** — 每流体抽取至阈值的 **2 倍** 才视为达标
5. **逐个停槽** — 单流体达标后其槽位自动切换至其他未达标流体
6. **全部达标关机** — 所有流体 ≥200% 时自动关闭全部钻机
7. **空余槽位随机补充** — 槽位数多于需抽取流体种类时，多余槽位随机分配

### 依赖组件

- `me_interface` — 读取 ME 网络流体存量
- `glasses`（AR 眼镜）— 实时 HUD 显示（可选，无眼镜时仅终端输出）
- `gt_machine` — 太空钻机控制

### 配置

```lua
local glassesInterval = 3    -- AR 眼镜刷新间隔（秒）
local checkInterval = 30     -- 维持检查间隔（秒）
local TARGET_RATIO = 2.0     -- 目标倍率（默认 200%）
```

### 流体配置

`FLUID_CONFIGS` 表按 **优先级从高到低** 排列，每项格式：

```lua
{注册名, 阈值, 行星参数, 气体参数, 显示名}
-- 例：{"deuterium", "1g", 6, 1, "氘"}
```

### AR 眼镜颜色

| 条件 | 颜色 |
|------|------|
| ME 断连 | 灰色 |
| <100% 阈值 且减少 | 红色 |
| <100% 阈值 且增加 | 橙色 |
| 100%~200% 且减少 | 黄色 |
| 100%~200% 且增加 | 绿色 |
| ≥200% | 白色 |

## pump_wiki.lua — 维基参考版

GTNH 中文维基来源脚本，功能包括：

- GPU 图形界面（触屏操作）
- 手动调整各流体优先级（TOP/UP/DOWN/BOTTOM）
- 手动调整权重（Weight）与目标值
- 支持 `target` / `always` / `off` 三种模式
- 配置文件持久化（`pump-config-v3.lua`）

> 此脚本为参考资料，推荐以 `pump.lua` 作为实际运行脚本。

## fluids_AR.lua — 纯监控脚本

独立的 AR 眼镜流体监控，仅显示 ME 网络中指定流体的存量与变化率，**不控制钻机**。适合只想看数据、手动管理钻机的场景。

## 使用方式

将 `pump.lua` 放入 OpenComputers 机器，终端运行：

```
pump
```

按 `Ctrl+C` 退出。
