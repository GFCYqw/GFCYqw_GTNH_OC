# Space Pump — 太空电梯流体监控与维持系统

GTNH 太空钻机（Space Pump）自动化管理脚本集，基于 OpenComputers。

## 文件说明

| 文件 | 类型 | 说明 |
|------|------|------|
| `pump.lua` | **主脚本** | AR 眼镜监控 + 缺额比例槽位分配 + 自动维持，推荐使用 |
| `pump_wiki.lua` | 参考 | GTNH 中文维基版，带 GPU 图形界面与手动控制 |
| `fluids_AR.lua` | 独立脚本 | 纯 AR 眼镜流体监控（无钻机控制） |
| `list_fluids.lua` | 占位 | 空文件 |

## pump.lua v2.0 — 主脚本

### 功能一览

1. **自动发现钻机** — 扫描 `gt_machine`，正则 `projectmodulepumpt[1-3]` 检测等级
2. **等级显示** — AR 眼镜和终端均显示 LV1/LV2/LV3 与总槽位数
3. **缺额比例分配** — 存量越低、缺额越大，分到的槽位越多（最大余数法）
4. **200% 目标** — 每流体抽取至阈值的 **2 倍** 才视为达标
5. **逐个停槽** — 单流体达标后自然退出分配，槽位自动转移
6. **全部达标关机** — 所有流体 ≥200% 时强制关闭全部钻机
7. **最小启停** — 仅槽位目标变化的机器才停止→改参数→启动

### 依赖组件

| 组件 | 必需 | 说明 |
|------|------|------|
| `me_interface` | ✅ | 读取 ME 网络流体存量 |
| `gt_machine` | ✅ | 太空钻机控制 |
| `glasses` | ❌ | AR 眼镜 HUD（无眼镜时仅终端输出） |

### 配置

```lua
glassesInterval = 3     -- AR 眼镜刷新间隔（秒）
checkInterval  = 30     -- 维持检查间隔（秒）
TARGET_RATIO   = 2.0    -- 目标倍率（默认 200%）
```

### 流体配置

`FLUID_CONFIGS` 表按优先级从高到低排列：

```lua
{注册名, 阈值, 行星参数, 气体参数, 显示名}
-- 例：{"deuterium", "1g", 6, 1, "氘"}
```

| 字段 | 说明 |
|------|------|
| 注册名 | ME 网络中流体的内部名称 |
| 阈值 | 告警下限，支持后缀 `k/m/g/t`，`-1` 表示忽略 |
| 行星参数 | 钻机 `planetType` |
| 气体参数 | 钻机 `gasType` |
| 显示名 | AR 眼镜和终端显示的中文名 |

## 核心算法：缺额比例槽位分配

### 整体流程

```
每 30 秒 ──→ assignSlots() ──→ 全部达标? ──是──→ shutdownAllMachines()
                  │                               强制关机
                  │ 否
                  ▼
         applySlotAssignments()
         仅变更槽位: 停止→改参数→启动
                  │
                  ▼
         drawDashboard()  刷新终端
```

### assignSlots() 四步

**Step 1 — 收集槽位**：遍历所有钻机所有槽位，扁平化。例：2台 LV2 × 4槽 = 8 槽。

**Step 2 — 计算缺额**：对每个流体（按配置顺序 = 优先级）：

```
target  = threshold × TARGET_RATIO     -- 200% 目标
deficit = target - amount              -- 缺额（仅 amount < target 时计入）
```

**Step 3 — 判断停机**：若所有流体 `amount ≥ target`，返回 `nil` 触发关机。

**Step 4 — 比例分配**（最大余数法）：

```
槽位数 = totalSlots × (该流体缺额 / 总缺额)
```

整数部分直接分配，小数部分按余数从大到小分配剩余槽位。

### 分配示例

> 2台 LV2 钻机（8槽），当前存量：

| 流体 | 当前 | 200%目标 | 缺额 | 占比 | 槽位 |
|------|------|----------|------|------|------|
| 液态空气 | 200M | 2.0G | 1.80G | 53% | **5** |
| 氘 | 500M | 2.0G | 1.50G | 44% | **3** |
| 氦-3 | 1.9G | 2.0G | 0.10G | 3% | **0** |
| 氚 | 3.5G | 2.0G | — | — | — |

> 氚已达标不参与分配；氦-3 缺额太小未分到槽位，待其他流体补足后自动获得槽位。

### applySlotAssignments() 最小启停

- 新分配与上次记录逐槽比对，仅目标变化的槽位才操作
- 按机器分组：同一台机器的多个槽位变更合并为一次启停
- 流程：`setWorkAllowed(false)` → 等停机 → `setParameters` → `setWorkAllowed(true)`

## AR 眼镜

### 颜色规则

| 存量 | 变化 | 颜色 | RGB |
|------|------|------|-----|
| 断连 | — | 灰色 | `128,128,128` |
| <100% 阈值 | 减少 | 红色 | `255,85,85` |
| <100% 阈值 | 增加 | 橙色 | `255,165,0` |
| 100%~200% | 减少 | 黄色 | `255,255,85` |
| 100%~200% | 增加 | 绿色 | `85,255,85` |
| ≥200% | — | 白色 | `255,255,255` |

```
液态空气: 1.2G (+50k/s)
氘: 800M (-10k/s)
氚: 1.5G (+30k/s)
...
```

## pump_wiki.lua — 维基参考版

GTNH 中文维基来源脚本（https://gtnh.huijiwiki.com/p/79342），功能包括：

- GPU 图形界面（触屏操作）
- 手动调整各流体优先级（TOP / UP / DOWN / BOTTOM）
- 手动调整权重（Weight）与目标值
- 支持 `target` / `always` / `off` 三种模式
- 配置文件持久化（`pump-config-v3.lua`）

> 此脚本为参考资料，推荐以 `pump.lua` 作为实际运行脚本。

## fluids_AR.lua — 纯监控脚本

独立的 AR 眼镜流体监控，仅显示指定流体存量与变化率，不控制钻机。

## 使用方式

将 `pump.lua` 放入 OpenComputers 机器，终端运行：

```
pump
```

按 `Ctrl+C` 退出。
